## 2.2.3

### Исправления

- Исправлена ошибка manifest merger от Android Gradle Plugin в модуле `core`: удалён устаревший атрибут `package=` и мёртвая, дублирующая декларация `LauncherActivity` (настоящая активность есть только в модуле `app`).
- Исправлено предупреждение CI (GitHub Actions) о запуске на устаревшем раннере Node.js 20 — обновлены версии `android-actions/setup-android` и `actions/upload-artifact`.

### Новое

- Добавлен полноценный английский перевод (`values-en`) вдобавок к русскому по умолчанию — теперь в приложении настоящая поддержка нескольких языков.
- Добавлен `Dockerfile`, чтобы можно было собрать APK в контейнере без установки Android SDK локально.

### Качество кода и инструменты разработки

- Включён Gradle Dependency Updates Plugin (`./gradlew dependencyUpdates`) для отслеживания устаревших зависимостей.
- Отложенный повтор синхронизации в `AlarmScheduler` переписан с `Handler`/`postDelayed` на Kotlin Coroutines.
- Строковые команды Dismiss/Snooze, которыми обмениваются телефон и часы, заменены на type-safe sealed class (`AlarmCommand`). Формат байт, уходящих через Data Layer, не изменился, поэтому релиз совместим со старой версией приложения на другом устройстве.

## 2.2.2

### Исправления ошибок

- **Фильтр источников будильника мог блокировать все будильники** на Android 11 и старше (или когда источник сигнала определить не удавалось, например без `showIntent`). Теперь при неизвестном источнике сигнал **разрешается**, а не блокируется.
- Исправлено пустое определение приложений-«Часов» на Android 11+: добавлена обязательная декларация `<queries>` для `ACTION_SET_ALARM` (видимость пакетов).
- Часы: пересборка расписания будильников после **перезагрузки** больше не рискует подвесить интерфейс — чтение из Data Layer теперь идёт в фоновом потоке.
- Часы: полноэкранное уведомление о будильнике теперь проверяет **разрешение на уведомления** перед показом и не рискует крашем, если разрешение отсутствует или отозвано.
- Часы: foreground-сервис для удержания процесса больше не крашится, если система отказала в `startForeground` (просто останавливает себя).
- Исправлен возможный **краш на часах с Android 8.0/8.1**: экран-лаунчер использовал тему, доступную только с Android 10.
- Исправлена невалидная константа важности канала уведомлений для будильника (использовалось несуществующее значение; теперь `IMPORTANCE_HIGH`).
- `NextAlarmReceiver` защищён от поддельных intent без корректного action.
- Добавлены недостающие русские переводы для двух строк, которые отображались на английском в остальном русском интерфейсе.

### Сборка, тесты и безопасность релиза

- Релизная сборка теперь **не даёт молча собраться с debug-подписью**, если `keystore.properties` отсутствует — если явно не передан флаг `-PallowDebugSignedRelease=true`.
- Добавлен CI-пайплайн на GitHub Actions: юнит-тесты, lint и сборка debug-APK при каждом push/PR.
- Добавлены юнит-тесты для логики синхронизации, вибрации и фильтра полуночи.
- Исправлены все ошибки lint и удалено ~20 неиспользуемых ресурсов (мёртвые строки, цвета, дублирующиеся файлы иконок); в baseline остались только предупреждения о версиях зависимостей, упирающиеся в текущую версию AGP.
- Обновлены AndroidX Core, AppCompat, Material, Play Services Wearable и Coroutines до последних версий, совместимых с текущим AGP.

## 2.2.0

### Источники будильников (телефон)

- На главном экране телефона: кнопка **«Источники будильников»** и строка-сводка, какие приложения сейчас разрешены для синхронизации на часы.
- Можно выбрать **несколько приложений** (чекбоксы), а не один переключатель «только Часы».
- По умолчанию синхронизируются только часы (Google Clock, MIUI Clock, Samsung Clock и др. — что найдётся на устройстве).
- Список формируется из нескольких источников:
  - приложения с intent `SET_ALARM` (классические «Часы»);
  - известные пакеты календаря (`com.android.calendar`, Google Calendar, Samsung, MIUI, `com.android.providers.calendar`);
  - **наблюдаемые** пакеты — сохраняются автоматически, когда приложение оказывается «следующим будильником» в системе;
  - **вручную добавленные** пакеты — кнопка «Добавить пакет» в диалоге.
- В списке показывается **имя приложения и пакет в скобках**, например: `Часы (com.google.android.deskclock)`.
- Если следующий системный сигнал пришёл от приложения **не из списка**, на часы он **не отправляется**; на экране телефона отображается пояснение.
- Старая настройка «только Часы» (`sync_clock_app_only`) **мигрирует** в новый список разрешённых пакетов при первом запуске.

### Фильтрация «полуночи»

- Сигналы ровно в **00:00** (часто календарь / напоминания) по-прежнему **не синхронизируются** на часы — фильтр всегда включён.
- На телефоне при таком случае показывается понятное сообщение вместо времени будильника.
- Очередь будильников (`AlarmQueueTracker`) использует ту же логику отсечения полуночи.

### Экран будильника на часах

- Кнопки **«Отклонить»** и **«Отложить»** — цветные: зелёная и красная (`ColoredAlarmButton`).
- Поведение dismiss/snooze не изменилось.

### Технические детали

- Новые классы: `AlarmSyncPrefs`, `AlarmSourceApps`, `AlarmSourceRecognizer`, `ColoredAlarmButton`.
- Источник будильника определяется по `showIntent.creatorPackage` (API 31+); на старых Android или без `creatorPackage` фильтр по приложению может не сработать.
- Android отдаёт только **один** «следующий» будильник на всю систему — это ограничение платформы, не приложения.
